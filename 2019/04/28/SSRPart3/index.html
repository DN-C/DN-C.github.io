面向小白的SSR搭建教程（三）如何在Windows/Linux/Android上使用SSR

这是面向小白的SSR搭建教程的最后一部分，也就是客户端的部署。我个人分别在Windows/Linux/Android上都有使用SSR，因此我会给出这三个平台的使用指南。

Windows 配置

Windows里面使用SSR是一个比较轻松愉快的过程，只需要下载配置好客户端，然后再简单配置一下chrome就可以了。

客户端下载与配置

请点击这里获取最新版本的windows客户端

点击上图红色箭头的地方，下载以zip为后缀名，带版本号的压缩包，解压即可看见单exe文件。打开，填入包括服务器ip地址，密码，加密方式等等参数，点击确定即可启用SSR。
为了节省服务器流量，选择pac模式，然后再将pac更新为GFWList。以上操作可以在右键单击软件系统托盘图标弹出来的菜单中设置。

ProxySwitchyOmega 配置

安装完毕后，打开插件的设置界面，新建场景模式，选择代理服务器模式，简单命名后完成创建。
进入新建的情景模式，将代理协议改为SOCKS5，代理服务器填部署的SSR中的本地IP（一般为127.0.0.1），端口同（一般为1080）。然后保存设置。下面是一个例子：

Android 配置

安卓的配置较为简单，来这里下载最新版本的apk，安装在安卓手机上，然后根据服务器端的配置填写安卓的相关参数即可。

需要注意的是，有些APP走SSR会出各种各样的问题，这个时候可以在APP内设置绕行模式，让这些APP不走SSR。具体设置较为简单，这里就不多说了。
